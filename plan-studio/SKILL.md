name: plan-studio
description: Draft or patch Plan documents for new or revised projects. Use when purpose, scope, constraints, phases, tasks, risks, review criteria, and deliverables must be structured without inventing new concepts. Do not use to write implementation code, detailed architecture, or execution logs.
---

# Goal

Produce a Plan document that is executable, scoped, and approval-ready.

# Inputs

- project name
- purpose
- in-scope items
- out-of-scope items
- constraints
- known risks
- existing Plan draft if this is a revision

# Workflow

1. Confirm the purpose in one sentence.
2. Preserve the user's vocabulary and boundaries.
3. Write these sections in order:
   - Overview
   - Goals
   - Scope
   - Constraints
   - Phases
   - Tasks
   - Risks
   - Review Criteria
   - Deliverables
4. Always split Scope into `In Scope` and `Out of Scope`.
5. Mark uncertainty as `Assumption:`.
6. If revising, perform patch-only edits instead of rewriting the full document.

# Outputs

- Plan draft or patch
- approval-readiness note when requested

# Guardrails

- No code.
- No new unapproved concepts, taxonomies, or process layers.
- No schedule or metric claims without evidence.
